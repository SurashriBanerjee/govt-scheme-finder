Phase 2: The Backend & Matching Logic (Days 4–8)
This is where you build the "brain" of the app. Use Python (Flask) or Node.js (Express)—whichever you are more comfortable with.

Step 3: Setup the Server Environment

Initialize your project (npm init for Node or setup a venv for Python).

Install the dependencies you will need: your server framework, dotenv (for API keys), and the official package for your AI layer (like @google/genai or openai).

Step 4: Build the Rule-Based Filter Engine

Write a basic function/endpoint (e.g., /api/match) that reads your schemes.json file.

Make it take a user profile object (JSON) and filter the array. For example:

JavaScript
// Hard filter logic sample
let matched = schemes.filter(s => user.income <= s.max_income && (s.gender_restriction === 'All' || s.gender_restriction === user.gender));
Test this endpoint using an API testing tool like Postman or Bruno. Input fake user profiles and make sure it correctly returns only the valid matching schemes.

Phase 3: The AI Chat Layer (Days 9–13)
Now we add the modern portfolio feature.

Step 5: Setup Context Injection

Create a /api/chat endpoint.

When a user types a question, your backend should grab the user's previously matched schemes, stringify that data, and inject it straight into the System Prompt of your LLM API call.

System Prompt Example: "You are a helpful grassroots assistant. Answer the user's question using ONLY the following verified scheme data: [Injected JSON Data]. Explain eligibility simply and break down jargon."

Step 6: Build the Profile Extractor (The Chat Shortcut)

Create a feature where if a user types a raw sentence instead of taking the quiz (e.g., "I'm a 12th pass girl from Delhi"), you pass that text to the LLM and strictly instruct it to return a clean JSON object matching your database keys. Feed that JSON directly into your Step 4 filter engine.

Phase 4: The Frontend UI (Days 14–17)
Keep it ultra-simple. Avoid heavy frameworks like React if you aren't already fluent in them. Plain HTML, CSS, and vanilla JavaScript work beautifully here.

Step 7: Build the Stream & Quiz Screens

Page 1: The entry landing page with two massive buttons: [Education] and [Healthcare].

Page 2: The quiz form. Use JavaScript to dynamically hide or show the education_level dropdown or healthcare_type options based on what they clicked on Page 1.

Step 8: Build the Results Feed & Chat Widget

Page 3: Display the matched schemes as clean, organized cards. Add the "Apply on Official Site" button redirecting to your exact official_url.

Embed a simple, fixed chat box UI on the side or bottom of the screen that connects to your /api/chat endpoint so they can instantly ask questions about the results.

Phase 5: Polish & Live Deployment (Days 18–20)
Step 9: Rigorous Testing & Disclaimers

Test 5 entirely different profiles (e.g., a high-income male student vs. a low-income female healthcare seeker) to ensure the matching logic is bulletproof.

Add your permanent safety disclaimer banner across the top or bottom of the application.

Deploy your backend and frontend to a free cloud hosting platform like Render, Railway, or Vercel.
